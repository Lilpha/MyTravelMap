# MyTravelMap

MyTravelMap은 사용자가 여행 중 찍은 사진과 영상을 업로드하면 지도에 마커로 표시하고, 마커를 클릭하면 해당 위치에서의 영상(또는 사진)을 재생하는 '여행 일기장' 웹 애플리케이션입니다. 단순한 갤러리가 아니라 "장소 기반의 기억 보관함"을 목표로 합니다.

## 주요 아이디어
- 사용자는 여행 중 촬영한 사진/영상을 업로드합니다.
- 업로드한 미디어는 위도/경도(혹은 지도 상에서 선택한 위치)와 함께 저장됩니다.
- 지도를 통해 사용자의 여행 경로와 방문지들을 시각적으로 확인할 수 있습니다.
- 마커를 클릭하면 해당 장소의 미디어(동영상/사진)를 재생하거나 보여주며, 간단한 메모(코멘트), 촬영일, 태그 등을 함께 표시합니다.

## 핵심 기능
- 장소 기반 업로드: 사진/영상 업로드 시 위치 정보를 연결하거나 지도에서 직접 위치를 선택.
- 지도 표시: Leaflet(오픈소스) 또는 Google Maps 중 선택하여 마커 렌더링.
- 미디어 재생: 마커 클릭 시 팝업 또는 모달에서 동영상 재생(지원 포맷: mp4, webm 등) 및 이미지 보기.
- 필터/태그: 날짜, 태그, 방문지 이름으로 필터링 가능.
- 간단한 일기 텍스트: 각 마커에 짧은 기록(감상, 날씨, 메모) 추가.

## 사용자 흐름
1. 사용자는 `업로드` 버튼으로 사진/영상을 선택합니다.
2. 업로드 폼에서 파일, 제목, 설명, 태그, 촬영일(선택), 위치(현재 위치 자동 입력 또는 지도에서 선택)를 입력합니다.
3. 저장 후 지도에 마커가 생성됩니다.
4. 마커를 클릭하면 팝업/모달에서 미디어가 재생되고 메모가 표시됩니다.

## 데이터 구조 예시 (JSON)
```
{
	"id": "uuid-1234",
	"title": "부산 송도 해변",
	"description": "저녁 노을과 함께한 서핑",
	"lat": 35.12345,
	"lng": 129.12345,
	"mediaType": "video", // "image" 또는 "video"
	"mediaUrl": "uploads/2025-11-14-songdo.mp4",
	"tags": ["바다", "노을"],
	"timestamp": "2025-11-14T17:32:00Z"
}
```

## 기술 스택(권장)
- 프론트엔드: HTML, CSS, JavaScript
- 지도 라이브러리: [Leaflet](https://leafletjs.com/) (간단/오프라인 친화적) 또는 Google Maps JavaScript API
- 미디어 저장: 초기에는 로컬 `uploads/` 폴더(간단한 프로토타입), 확장 시 백엔드(Express/Flask 등) + 클라우드 스토리지(S3 등)
- 데이터 저장: 로컬 JSON 파일 또는 간단한 DB(SQLite, lowdb). 확장 시 PostgreSQL 등

## 파일 구조(현재 워크스페이스 기반 권장)
- `index.html` - 메인 페이지(지도 + 업로드 UI)
- `css.css` / `Prac/main.css` - 스타일
- `Prac/index.html`, `Prac/issue.html` - 실험/프로토타입 페이지
- `Prac/img/`, `Prac/img/icon/` - 아이콘 및 이미지 리소스
- `uploads/` (추가) - 업로드된 미디어 저장 폴더

> 참고: 현재 워크스페이스에 `css.css`, `index.html`, `Prac/` 하위 파일들이 있으므로 `index.html`에 지도를 추가하고 업로드 UI를 연결하는 방식으로 시범 구현을 시작하세요.

## 로컬에서 실행(개발용)
- 간단한 정적 서버(권장): Python 내장 서버 사용
```powershell
python -m http.server 8000
```
- 또는 VS Code의 `Live Server` 확장 사용.
- 브라우저에서 `http://localhost:8000` 열기.

## 개발 노트 / 구현 팁
- 마커 팝업 대신 모달을 사용하면 동영상 재생(자동 재생·크기 제어)이 더 좋습니다.
- 비디오 포맷은 브라우저 호환성 고려: mp4(h.264) 권장, webm 대체.
- 로컬에서는 CORS/파일 경로 문제로 미디어 재생이 안되는 경우가 있으니 웹서버로 제공해야 합니다.
- 모바일 반응형: 지도 상의 팝업 대신 하단 슬라이드업 패널을 고려.

## 향후 확장 아이디어
